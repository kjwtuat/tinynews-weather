# 09. tinynews-weather

**"AI 기상캐스터가 매일 날씨를 브리핑해주는 웹 서비스"**

기상청 RSS 서비스가 종료(API 전환)됨에 따라, 대체 수단을 마련하고 AI를 활용해 시각장애인이나 운전자도 듣기 편한 **구어체(TTS용) 날씨 대본을 자동 생성**하도록 파이프라인을 구축한 프로젝트입니다.

## 주요 기능 및 작업 내역

### 1. 날씨 데이터 및 기사 수집 로직 (`fetchWeather.js`)
* **구글 뉴스 RSS 연동**: '오늘 날씨' 키워드로 구글 뉴스 RSS를 검색하여 최신 기사 10개를 가져옵니다 (`rss-parser` 사용).
* **주요 도시 날씨 수집**: `wttr.in` API를 사용하여 서울, 대전, 대구, 부산, 광주, 제주의 오늘의 최고/최저 기온 및 현재 습도 데이터를 JSON 형태로 수집합니다.

### 2. Gemini AI를 활용한 기상캐스터 대본 자동 생성
* 수집된 뉴스 기사 요약본과 도시별 기온 데이터를 **Google Gemini API (`gemini-2.5-flash`)**에 프롬프트와 함께 전달합니다.
* AI 모델에게 **"친절하고 활기찬 AI 기상캐스터"** 역할을 부여하여, TTS로 들었을 때 아주 자연스러운 구어체 대본을 작성하도록 지시했습니다.
* 대본은 항상 다음 4가지 파트로 나뉘어 JSON 구조로 반환되도록 엄격히 설계되었습니다.
  1. 전국 날씨 요약
  2. 주의 사항 (폭염, 미세먼지 등)
  3. 주요 도시 날씨 (기온 및 습도 상세 브리핑)
  4. 내일 날씨 전망

### 3. 데이터 저장 및 자동화 파이프라인
* 생성된 날씨 대본 데이터는 KST(한국 표준시) 기준 날짜 파일(예: `2026-07-22.json`)로 `public/data/` 폴더에 자동 저장됩니다.
* 스토리지가 무한정 늘어나지 않도록 **최근 7일 치의 데이터만 유지**하고 오래된 파일은 자동으로 삭제하는 로직을 구현했습니다.
* 조회 가능한 날짜 목록은 `public/data/index.json`에 업데이트됩니다.

### 4. 프론트엔드 및 배포 환경 설정
* **Vite**를 기반으로 빠르고 가벼운 프론트엔드 환경을 세팅했습니다. (`vite.config.js`, `main.js`, `style.css` 등)
* GitHub Actions를 연동하기 위해 `.github/workflows/deploy.yml`을 작성하여, 정해진 시간이나 코드가 푸시되었을 때 **데이터 갱신 및 GitHub Pages 자동 배포**가 이루어지도록 CI/CD 파이프라인을 구축했습니다.
